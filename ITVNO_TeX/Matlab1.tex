\section{Рассчет потока вектора через поверхность в пакете Matlab}
Пусть в точке (0,0,0) находится заряд $Q$. Задача: найти поток вектора $\mathbf{E}$ через поверхность, заданную функцией $f(x,y,z)=z+ax^2+by^2-H=0$ при $z>0$. 

Поток вектора задается формулой:
\begin{equation}
\label{Potok}
\Phi=	\iint \mathbf{E}\mathbf{n} \mathrm{d} S. 
\end{equation}
 
 Напряженность задается формулой:
 \begin{equation}
 \mathbf{E}=\frac{Q}{r^3} \mathbf{r}
 \end{equation}
   
    Вектор нормали можно рассчитать по формуле:
    \begin{equation}
     \mathbf{n}= \frac{\nabla f(x,y,z)}{|\nabla f(x,y,z)|}
    \end{equation}

     \section{Разработка программ с использованием репозиториев}
     Ни один серьезный проект невозможно разработать без двух очень важных моментов:
     \begin{itemize}
     	\item Организация эффективного взаимодействия участников группы.
     	\item Контроль версий проекта, с указанием что конкретно и когда было сделано.
     \end{itemize}
     Впринципе, можно справится с этими проблемами уже известными вам способами. Можно вести часть документов в GoogleDocs, можно постоянно рассылать по электронной почте или в каком-то из мессенджеров новые версии проекта. Можно сохранять разные версии в разных папках на компьютере. Но, по мере усложнения проекта, делать это становится все затруднительнее. Чтобы справиться была создана  распределённая система управления версиями Git (произн. «гит»). 
     
     
  Скачать его можно здесь: \href{http://git-scm.com}{http://git-scm.com}. При установке пока просто нажимайте кнопку "далее".

Для многих это будет непривычно, но не так много комманд, которые вам понадобятся для начала работы. Небольшой интерактивный туториал можно пройти вот здесь: \href{https://try.github.io}{https://try.github.io}. 
   Русский язык работает с git bash плохо, поэтому следите, чтобы полный путь папок, где   хранятся ваши файлы, не содержал русскоязычных имен папок (ваших папок пользователя   это тоже касается).
   
   Книга по Git на русском языке: \href{http://habrahabr.ru/post/150673/}{habrahabr.ru/post/150673/}

   \section{Метод Рунге-Кутты 4 степени. }
   
   В методе Рунге-Кутты 4 степени значение переменной в каждой следующей точке задается следующим соотношением:
   \begin{equation}
   	\rho(\varphi+\Delta\varphi)=\rho(\varphi)+(f(\varphi_1, \rho_1)+2f(\varphi_2, \rho_2)+2f(\varphi_3, \rho_3)+f(\varphi_4, \rho_4))\Delta\varphi/6.
   \end{equation}
   Здесь $f=\pm\frac{\rho^2}{M}\sqrt{2m(E-U^{\mathrm{eff}} (\rho))}$. Значение параметров $\varphi_i, \rho_i$ определяется выражениями:
   $$
   \varphi_1=\varphi, \rho_1=\rho(\varphi)
   $$
       $$
       \varphi_2=\varphi+\Delta\varphi/2, \rho_2=\rho(\varphi)+f(\varphi_1, \rho_1)\Delta\varphi/2
       $$
       $$
     \varphi_3=\varphi+\Delta\varphi/2, \rho_3=\rho(\varphi)+f(\varphi_2, \rho_2)\Delta\varphi/2
      $$
             $$
             \varphi_4=\varphi+\Delta\varphi, \rho_4=\rho(\varphi)+f(\varphi_3, \rho_3)\Delta\varphi
             $$
Таким образом на каждом шаге мы вычисляем значение функции несколько раз, но и сходимость такой процедуры тоже более высокая. Не требуется считать много шагов. Теперь напишем программу, которая считала бы этим методом, но при этом учитывала бы проблему знака.
